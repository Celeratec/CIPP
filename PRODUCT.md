# Product

## Register

product

## Users

MSP technicians managing multiple client Microsoft 365 tenants daily. They're juggling dozens of clients, context-switching constantly, and often working under time pressure. They know M365 deeply — they don't need training wheels, they need speed.

## Product Purpose

Manage365 is a multi-tenant M365 administration portal that lets MSPs manage users, teams, SharePoint, Exchange, security, Intune, and more across all their clients from a single interface. Success looks like: a technician who used to spend 20 minutes clicking through three Microsoft portals now completes the same task in 2 minutes without leaving Manage365.

## Brand Personality

Fast, focused, technical.

The UI should feel like a power tool — responsive, information-dense, keyboard-friendly. No marketing language, no friendly mascots, no "getting started" hand-holding. Assume expertise.

## Anti-references

- **Azure Portal** — Overwhelming navigation, deeply nested menus, slow page loads, context constantly lost
- **Generic SaaS** — Gradient hero sections, stock photography, marketing copy masquerading as UI, feature announcements nobody asked for
- **Old-school admin tools** — Cramped data grids, 11px fonts, Windows XP-era iconography, forms that feel like tax software
- **Over-designed apps** — Excessive white space, animations that slow you down, style over substance, pretty but inefficient

Reference: Raycast — snappy, dark, power-user focused. Every pixel serves a purpose.

## Design Principles

1. **Speed over ceremony** — Every interaction should feel instant. Prefetch data, avoid confirmation dialogs for reversible actions, never make users wait for something that can happen in the background.

2. **Density with clarity** — Pack information tight without cramping. Data tables should be scannable, not endlessly scrollable. Show what matters at a glance; let details expand on demand.

3. **Expert confidence** — Assume the user knows M365. Provide keyboard shortcuts, bulk actions, and direct paths. Skip the wizards when a single well-designed form will do.

4. **Multi-tenant fluent** — Every screen acknowledges the user is managing many clients. Tenant switching must be instant. Never lose context when moving between clients.

5. **Opinionated defaults** — Make the right choice easy. Pre-fill sensible values, surface recommended actions, reduce decisions that don't matter.

## Accessibility & Inclusion

Standard best practices: sufficient color contrast, keyboard navigation, screen reader compatibility, and focus indicators. Don't rely on color alone to convey status — pair with icons or text. Support prefers-reduced-motion for users who need it.
